# Летний Розыгрыш 2026 — сайт ивента

Сайт для совместного ивента трёх стримеров: **PariPariKai × Ilyich116 × d1psy_off**.

## Файлы

- **`SHEETS_INSTRUCTIONS.md`** — инструкция для Codex по созданию Google Sheets и Google Form
- **`tickets.html`** — публичная страница с таблицей билетов и призовой зоной (URL даётся в чатах по команде `!ивент`)
- **`wheel.html`** — приватная страница с колесом фортуны для розыгрыша (только для стримеров, защищена паролем)

## Шаги развёртывания

### 1. Подготовить Google Sheets
Передай файл `SHEETS_INSTRUCTIONS.md` Codex'у. Он создаст таблицу, форму и опубликует листы как CSV. На выходе получишь **3 ссылки**:
- `TICKETS_CSV_URL` (лист Tickets)
- `CONFIG_CSV_URL` (лист Config)
- `VIEWERS_CSV_URL` (лист Viewers)

### 2. Вставить ссылки в HTML
В обоих файлах в начале `<script>` блока замени плейсхолдеры на реальные URL:

**`tickets.html`** (строки 16–18):
```js
const TICKETS_CSV_URL = "..."; // лист Tickets
const CONFIG_CSV_URL  = "..."; // лист Config
const VIEWERS_CSV_URL = "..."; // лист Viewers
```

**`wheel.html`** (строки 20–21):
```js
const TICKETS_CSV_URL = "..."; // лист Tickets
const VIEWERS_CSV_URL = "..."; // лист Viewers
```

### 3. Установить пароль на колесо
В `wheel.html` нужно вставить SHA-256 хэш пароля (а не сам пароль). Открой консоль браузера на любой странице и выполни:

```js
crypto.subtle.digest('SHA-256', new TextEncoder().encode('твой_пароль_сюда'))
  .then(b => console.log(Array.from(new Uint8Array(b)).map(x=>x.toString(16).padStart(2,'0')).join('')))
```

Скопируй полученную строку и вставь в `wheel.html` (строка 22):
```js
const WHEEL_ACCESS_HASH = "вставь_хэш_сюда";
```

### 4. Создать новый репозиторий и задеплоить на GitHub Pages
- Создай новый репозиторий (можно `summer-raffle-2026`)
- Положи туда `tickets.html` и `wheel.html`
- Settings → Pages → Source: `main` branch, root
- Дождись деплоя

URL будут выглядеть так:
- `https://<твой-юзернейм>.github.io/summer-raffle-2026/tickets.html`
- `https://<твой-юзернейм>.github.io/summer-raffle-2026/wheel.html`

### 5. Настроить команду `!ивент` в чате
В чатах всех трёх стримеров добавь команду, выдающую ссылку на `tickets.html`.

Ссылку на `wheel.html` **никому не сообщай** — только трое стримеров знают URL и пароль.

## Как пользоваться

### Стримеры выдают билеты
Открыть Google Form (сохранить как закладку на телефоне):
1. Ввести ник зрителя
2. Выбрать источник билетов (рулетка, активность, топ чатерс и т.д.)
3. Ввести количество
4. Выбрать себя (стример)
5. Отправить

Данные автоматически попадут в лист `Tickets` через формулы. На сайте обновятся через ~1 минуту (или сразу при перезагрузке).

### Зрители смотрят свои билеты
Открывают `tickets.html` → видят:
- Текущий приз месяца (бронза → серебро → золото) с таймером
- Превью всех трёх месяцев лета
- Топ-3 на подиуме
- Поиск по нику
- Свою позицию и количество билетов

### Розыгрыш в конце месяца
В последний день месяца (30 июня / 31 июля / 31 августа):
1. Один из стримеров открывает `wheel.html`, вводит пароль
2. Колесо подгружает текущие билеты
3. Запускает розыгрыш на вылет → определяется победитель
4. Победитель получает приз (Озон-карта + скины CS)

## Что управляется через Google Sheets без правки кода

В листе `Config` можно менять:
- Баннер шапки (URL картинки)
- Название и подзаголовок ивента
- Суммы призов по месяцам
- Даты розыгрышей
- Включение/выключение таймера (`raffle_active`)
- Принудительный сдвиг тира приза (`current_prize_override`)
- Аватарки и ссылки на каналы стримеров

Автоматически по дате:
- Какой месяц "текущий" (июнь / июль / август)
- Какой тир показывать (bronze / silver / gold)
- На какую дату идёт таймер
