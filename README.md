## Порядок развёртывания

### 1. Перестроить Google Sheet через Codex

Отдай `SHEETS_INSTRUCTIONS_v2.md` Codex'у. Инструкция говорит:
- Очистить лист `Tickets`
- Создать новую структуру с 53 колонками (B–AQ для источников × 3 стримера + AR–BA для итогов)
- Обновить опции в Google Form (добавить эмодзи 🟡/⚪ для визуальной маркировки типа)
- Сохранить существующие CSV-ссылки

Старая Google Form останется работать — Codex только обновит выпадающий список источников.

### 2. Обновить файлы сайта

Из старого репозитория **скопируй CSV-ссылки и хэш пароля**:
- `TICKETS_CSV_URL`, `CONFIG_CSV_URL`, `VIEWERS_CSV_URL` — в `index.html` и `tickets.html`
- `TICKETS_CSV_URL`, `VIEWERS_CSV_URL`, `WHEEL_ACCESS_HASH` — в `wheel.html`

В шаблонах эти значения помечены как `PASTE_YOUR_..._HERE`.

### 3. Замени файлы в репозитории

**Структура репозитория должна быть такой:**
```
/
├── index.html       ← НОВЫЙ ЛЕНДИНГ (заменяет старую index.html)
├── tickets.html     ← ТАБЛИЦА БИЛЕТОВ (была index.html, теперь переименована)
├── wheel.html       ← КОЛЕСО (как было)
```

То есть:
1. **Удалить** старый `index.html` (это была таблица билетов)
2. **Заменить** на новый `index.html` из этой папки (это лендинг)
3. **Добавить** новый `tickets.html` — теперь это таблица билетов
4. **Заменить** `wheel.html` на новую версию из этой папки

### 4. Команда `!ивент` в чатах

Раньше выдавала ссылку на `index.html` (таблица билетов). Теперь должна выдавать **ссылку на тот же index.html**, но это уже лендинг, с которого зрители могут перейти на таблицу билетов через CTA-кнопку.

Альтернативно — можно сделать **две команды**:
- `!ивент` → главная (лендинг с правилами)
- `!билеты` → сразу таблица билетов (`tickets.html`)

Так зрители смогут быстро попасть туда, куда им надо.

### 5. Деплой

После замены файлов в локальной папке репозитория:

2. Попроси Claude Code / Composer закоммитить изменения и запушить

### Известные особенности

- **Серебро отображается как `27/100`** на странице билетов — это нормально, показывает упёрся ли зритель в кап
- **Если зритель упёрся в 100 серебра** — формула SUMIFS в Sheets всё равно складывает все начисления (например 105), а итог обрезается через `MIN(100, ...)`. То есть стример может видеть в таблице `silver_roulette_ppk = 105`, но в `silver_ppk = 100`. Так и должно быть.
- **На раскрытой карточке** показано серебро с дробной частью `/100`, а золото без — потому что у золота нет лимита.
