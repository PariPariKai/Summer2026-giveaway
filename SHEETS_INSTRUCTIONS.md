# Google Sheets для "Летнего Розыгрыша 2026"

Создай новую Google Таблицу с названием **"Summer Raffle 2026 — Tickets"**.

В таблице должно быть **3 листа** (вкладки) с точными именами:
1. `Tickets` — учёт билетов зрителей
2. `Viewers` — база зрителей (ники + аватарки)
3. `Config` — настройки сайта

Дополнительно нужно создать **Google Form** для удобного ввода билетов стримерами с телефона/компьютера во время стрима. Форма пишет в отдельный скрытый лист `FormResponses`, а лист `Tickets` агрегирует данные оттуда через формулы.

После создания нужно **опубликовать каждый из трёх листов** как CSV (File → Share → Publish to web → выбрать конкретный лист → формат CSV) и сохранить полученные URL — они пойдут в HTML-файлы сайта.

## Лист 1: `Tickets`

Этот лист — главный. Он показывает у каждого зрителя сколько билетов он набрал по каждому источнику и общую сумму.

### Структура колонок

| Колонка | Заголовок | Описание |
|---------|-----------|----------|
| A | `nick` | Ник зрителя (вводится один раз) |
| B | `subs_donates` | Подписки + донаты (1 билет = 1 подписка или донат 150₽) |
| C | `points_buy` | Покупка билетов за баллы канала |
| D | `roulette` | Билеты с рулетки (7% шанс) |
| E | `activity_wins` | Победы в активностях (Шпион, Пирамида, викторины, Гартик и т.д.) |
| F | `predictions` | Билеты с прогнозов "да/нет" (3/2/1) |
| G | `top_chat_week` | Топ-3 чатеров за неделю (3/2/1) |
| H | `top_chat_month` | Топ-3 чатеров за месяц (10/7/3) |
| I | `top_view_week` | Топ-3 онлайна за неделю (3/2/1) |
| J | `top_view_month` | Топ-3 онлайна за месяц (10/6/4) |
| K | `top_donate_week` | Топ-3 донатеров за неделю (10/5/3) |
| L | `top_donate_month` | Топ-3 донатеров за месяц (20/10/5) |
| M | `top_clip_month` | Топ-1 клип месяца (5 билетов) |
| N | `twitch_points` | Билеты за баллы Твича (1 за стрим) |
| O | `collab_bonus` | Коллаб-бонус (1 билет за активность в общем чате коллаба) |
| P | `tickets` | **ИТОГО** — сумма всех билетов |
| Q | `avatar_url` | Аватарка (подтягивается из `Viewers` через VLOOKUP) |

### Формулы

**Колонка P (`tickets`)** — итоговая сумма, начиная со строки 2:
```
=IFERROR(SUM(B2:O2), 0)
```
Эту формулу нужно протянуть вниз минимум на 500 строк (можно использовать `ARRAYFORMULA` чтобы не тянуть).

Вариант через ARRAYFORMULA (поставить в P1 вместо заголовка, или в P2 с ARRAYFORMULA от P2:P):
```
=ARRAYFORMULA(IF(A2:A="",,IFERROR(MMULT(N(B2:O), TRANSPOSE(COLUMN(B1:O1)^0)), 0)))
```
(Если ARRAYFORMULA сложна — оставь обычную SUM и протяни вниз на 500 строк.)

**Колонка Q (`avatar_url`)** — VLOOKUP из листа `Viewers`:
```
=IFERROR(VLOOKUP(A2, Viewers!A:B, 2, FALSE), "")
```
Аналогично протянуть вниз.

### Источник данных в B–O колонках

Колонки B–O **заполняются автоматически** из листа `FormResponses` через SUMIFS. То есть стример **не редактирует лист Tickets вручную** — он только заполняет Google Form, а данные собираются автоматически.

Формула для ячейки B2 (`subs_donates` для первого зрителя):
```
=IFERROR(SUMIFS(FormResponses!D:D, FormResponses!B:B, A2, FormResponses!C:C, "subs_donates"), 0)
```

Где:
- `FormResponses!D:D` — колонка с количеством билетов в логе формы
- `FormResponses!B:B` — колонка с ником зрителя в логе формы
- `FormResponses!C:C` — колонка с типом источника в логе формы
- `A2` — ник текущего зрителя из листа Tickets

**Для каждой колонки B–O меняется только значение третьего аргумента** (тип источника). Используй эти ключи (точное совпадение, без пробелов):

| Колонка | Ключ источника |
|---------|----------------|
| B | `subs_donates` |
| C | `points_buy` |
| D | `roulette` |
| E | `activity_wins` |
| F | `predictions` |
| G | `top_chat_week` |
| H | `top_chat_month` |
| I | `top_view_week` |
| J | `top_view_month` |
| K | `top_donate_week` |
| L | `top_donate_month` |
| M | `top_clip_month` |
| N | `twitch_points` |
| O | `collab_bonus` |

**Альтернатива через ARRAYFORMULA** (одна формула для всей колонки, ставится в B2):
```
=ARRAYFORMULA(IF(A2:A="",,IFERROR(SUMIFS(FormResponses!D:D, FormResponses!B:B, A2:A, FormResponses!C:C, "subs_donates"), 0)))
```
Это удобнее — не нужно тянуть формулы вручную. Применить ко всем колонкам B–O, меняя ключ источника.

### Ручной ввод (без формы)

Если стример хочет вписать билеты **напрямую без формы** — для этого можно использовать сам лист `FormResponses` (просто добавить строку: timestamp, ник, тип источника, количество). Лист `Tickets` подхватит автоматически.

Это лучше, чем редактировать `Tickets` напрямую, потому что:
- Сохраняется история всех начислений
- Можно потом отфильтровать "кто и когда выдал билет"
- Не ломается формула в `Tickets`

## Лист 2: `Viewers`

База всех зрителей с их аватарками.

### Структура колонок

| Колонка | Заголовок | Описание |
|---------|-----------|----------|
| A | `nick` | Ник зрителя (точно как пишется в чате Twitch) |
| B | `avatar_url` | URL аватарки (прямая ссылка на картинку, например `https://...png`) |

### Заполнение

Заполняется вручную по мере появления новых зрителей. Если зритель появился в `Tickets`, но его нет в `Viewers` — на сайте у него будет показываться placeholder (инициалы).

**Рекомендация:** добавляй зрителей сюда **до** того, как они получат первый билет, чтобы аватарка сразу была.

## Лист 4 (скрытый): `FormResponses`

Этот лист **автоматически создаётся Google Form** когда ты привязываешь форму к таблице. Не редактируй его структуру вручную — Google сам её создаст.

Структура которую создаст форма:
| Колонка | Заголовок |
|---------|-----------|
| A | `Timestamp` (создаётся автоматически) |
| B | `Ник зрителя` |
| C | `Источник билетов` (одно из значений-ключей выше) |
| D | `Количество билетов` |
| E | `Стример` (кто выдал — опционально, для статистики) |

После создания формы можно **скрыть этот лист** (ПКМ по вкладке → Hide sheet), чтобы он не мешал.

## Google Form (важно!)

Создай Google Form со следующими полями:

1. **Ник зрителя** — короткий текстовый ответ, обязательное поле
2. **Источник билетов** — выпадающий список (dropdown), обязательное поле:
   - `subs_donates` (подписка / донат 150₽)
   - `points_buy` (покупка за баллы канала)
   - `roulette` (рулетка)
   - `activity_wins` (победа в активности)
   - `predictions` (прогноз да/нет)
   - `top_chat_week` (топ чатерс недели)
   - `top_chat_month` (топ чатерс месяца)
   - `top_view_week` (топ онлайн недели)
   - `top_view_month` (топ онлайн месяца)
   - `top_donate_week` (топ донатер недели)
   - `top_donate_month` (топ донатер месяца)
   - `top_clip_month` (топ клип месяца)
   - `twitch_points` (баллы Твича)
   - `collab_bonus` (коллаб-бонус)
3. **Количество билетов** — короткий текстовый ответ с валидацией "число", обязательное поле
4. **Стример** — выпадающий список:
   - PariPariKai
   - Ilyich116
   - d1psy_off

Затем привяжи форму к этой таблице (Responses → Link to Sheets → Select existing spreadsheet → выбрать "Summer Raffle 2026 — Tickets"). Это создаст лист `FormResponses` (или с похожим именем — переименуй в `FormResponses` если нужно).

**Сохрани короткую ссылку на форму** — её удобно сохранить как закладку на телефоне и заполнять во время стрима.

## Публикация листов как CSV

После того как всё создано и заполнено, **опубликуй каждый из трёх листов** (`Tickets`, `Viewers`, `Config`) как CSV:

1. File → Share → Publish to web
2. В первом dropdown выбери конкретный лист (НЕ "Entire Document")
3. Во втором dropdown выбери "Comma-separated values (.csv)"
4. Нажми Publish
5. Скопируй полученный URL — он понадобится для HTML-файлов сайта

Должно получиться **3 CSV URL**:
- `TICKETS_CSV_URL` — лист Tickets
- `VIEWERS_CSV_URL` — лист Viewers
- `CONFIG_CSV_URL` — лист Config

**Лист FormResponses не публикуй** — он содержит сырые данные, на сайте не нужен.

---

## Права доступа

- Сама Google Таблица: **редактирование** должно быть у всех троих стримеров (Share → Add people)
- Google Form: **редактирование** тоже у всех троих (чтобы могли менять список источников при необходимости)
- Опубликованные CSV: публичные по ссылке (это нормально — данные о билетах не секретные)
